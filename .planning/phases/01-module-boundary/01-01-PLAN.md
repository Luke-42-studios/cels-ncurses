**include/cels-ncurses/tui_ncurses.h** -- Public component types:
- Include guard: CELS_NCURSES_TUI_NCURSES_H
- Includes: `<cels/cels.h>` (for CEL_Component macro, cels_entity_t), `<stdbool.h>`
- Module declaration: `extern cels_entity_t NCurses;` and `extern void NCurses_init(void);`

- NCurses_WindowState component via CEL_Component macro: width (int), height (int), running (bool), actual_fps (float), delta_time (float)
- NCurses_InputState component via CEL_Component macro: Copy ALL fields from the existing TUI_InputState struct in tui_input.h verbatim (axis_left, button_accept, button_cancel, key_tab, key_shift_tab, key_home, key_end, key_page_up, key_page_down, key_backspace, key_delete, key_number, has_number, key_function, has_function, raw_key, has_raw_key, mouse_x, mouse_y, mouse_button, mouse_pressed, mouse_released, mouse_left_held, mouse_middle_held, mouse_right_held). Also define the TUI_MouseButton enum before it (copy from tui_input.h).
- Forward-declare NCurses_Layer as an empty CEL_Component (Phase 4 will fill it in)

- Include guard: CELS_NCURSES_TUI_INTERNAL_H

  - `void ncurses_register_window_observer(void);`

IMPORTANT: Use the exact CELS v0.5 API from 01-RESEARCH.md. CEL_Component (uppercase), not cel_component. Do not include `<flecs.h>` in public headers -- flecs internals stay in .c files.

**src/ncurses_module.c** -- Module definition following 01-RESEARCH.md Pattern 1 and Example 3:
- Include tui_ncurses.h (for component types) and tui_internal.h (for registration function declarations)
- Include `<flecs.h>` (OK in .c files)
- CEL_Module(NCurses) body that:
  1. Registers NCurses_WindowConfig via cels_ensure_component(&NCurses_WindowConfig_id, "NCurses_WindowConfig", sizeof(NCurses_WindowConfig), CELS_ALIGNOF(NCurses_WindowConfig))
  2. Registers NCurses_WindowState the same way
  3. Registers NCurses_InputState the same way
  4. Calls ncurses_register_window_observer() -- EcsOnSet for terminal init
  5. Calls ncurses_register_window_remove_observer() -- EcsOnRemove for clean shutdown
  6. Calls ncurses_register_input_system() -- OnLoad: read getch each frame
  7. Calls ncurses_register_frame_systems() -- PreStore/PostFrame: begin/end frame
- The module does NOT create any entities (per Pitfall 2 in RESEARCH.md)
- The module does NOT call tui_frame_init() -- frame init is handled inside the window observer (terminal must be active first)

**CMakeLists.txt** -- Update the INTERFACE library:
- In target_sources: Replace `src/tui_engine.c` with `src/ncurses_module.c`. Keep all other source files unchanged (tui_window.c, tui_input.c, tui_frame.c, graphics/*, layer/*)
- Do NOT add src/widgets/tui_widgets.c to target_sources (it will be deleted in Plan 02)
- In target_link_libraries: Remove `cels-widgets` from the INTERFACE link. Keep: ${CURSES_LIBRARIES}, ${PANEL_LIBRARY}, cels, m
- Keep the draw_test executable target unchanged for now (Plan 03 will rebuild it)
- Keep all compile definitions unchanged (_XOPEN_SOURCE=700, _DEFAULT_SOURCE, CELS_HAS_ECS)

NOTE: The build will NOT succeed after this plan alone because tui_window.c / tui_input.c / tui_frame.c still have the old provider-registration API.
