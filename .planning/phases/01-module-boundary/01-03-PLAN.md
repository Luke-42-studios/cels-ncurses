**Rewrite examples/minimal.c** as the Phase 1 validation example, following 01-RESEARCH.md Pattern 5 (Consumer Application Usage):

```c
// Phase 1 validation: NCurses module loads, window entity opens terminal
#include <cels/cels.h>
#include <cels-ncurses/ncurses.h>

// Root composition: create window entity with config
CEL_Composition(World) {
    (void)props;

    cel_entity(.name = "Window") {
        cel_has(NCurses_WindowConfig,
            .title = "Phase 1 Test",
            .fps = 30,
            .color_mode = 0  // auto
        );
    }
}

cels_main() {
    cels_scope {
        cels_world->title = "Phase 1 Module Test";
    }

    // Load NCurses module (registers components, systems, observers)
    NCurses_init();

    cels_register(World);

    cels_session(World) {
        while (cels_running()) {
            cels_step(0);
        }
    }
}
```

Adjust the exact syntax based on what CELS v0.5 actually provides. The key elements:
- `#include <cels/cels.h>` and `#include <cels-ncurses/ncurses.h>` -- two includes
- NCurses_init() to load the module
- Entity with NCurses_WindowConfig component
- cels_main / cels_session / cels_running / cels_step loop

If `cels_main()`, `cels_session()`, or `cels_running()` don't exist in the current CELS API, fall back to the pattern used in the existing examples (CEL_Build + whatever main loop pattern CELS provides).
