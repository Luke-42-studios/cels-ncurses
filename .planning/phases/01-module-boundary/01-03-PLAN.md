  <name>Task 2: Rewrite minimal.c with new NCurses module API</name>

  <action>
Rewrite `examples/minimal.c` to use the new entity-component API. This is the reference implementation that proves Phase 1 works.

The example MUST match the API decisions from CONTEXT.md:

```c
#include <cels/cels.h>
#include <cels-ncurses/ncurses.h>

CEL_Composition(World) {
    NCursesWindow(.title = "Phase 1 Test", .fps = 30) {}
}

cels_main() {
    cels_register(NCurses);
    cels_session(World) {
        while (cels_running()) cels_step(0);
    }
}
```

Key API patterns demonstrated:
1. `#include <cels/cels.h>` -- CELS framework (developer includes separately)
2. `#include <cels-ncurses/ncurses.h>` -- NCurses module (single umbrella include)
3. `CEL_Composition(World)` -- Root composition (NOT CEL_Build, NOT CEL_Root)
4. `NCursesWindow(.title = "X", .fps = 30) {}` -- Call macro from CEL_Define, creates entity with NCurses_WindowConfig. The `{}` is the composition body (empty here -- no children).
5. `cels_main()` -- Modern entry point (NOT CEL_Build)
6. `cels_register(NCurses)` -- Module registration (expands to NCurses_register() which Phase 0 adds as alias for NCurses_init()). NOTE: Until Phase 0 is complete, this may need to temporarily use NCurses_init() directly. Check if NCurses_register exists (Phase 0 may already be done). If not, use NCurses_init() with a TODO comment.
7. `cels_session(World)` -- Opens the developer loop
8. `while (cels_running()) cels_step(0)` -- Standard CELS main loop

Do NOT include:
- `(void)props` or `(void)cel` in the composition body -- `cel` is already `_CEL_UNUSED`
- `cel_entity(.name = "Window")` -- use the NCursesWindow call macro instead
- `cel_has(NCurses_WindowConfig, ...)` directly -- the NCursesWindow composition does this internally
- Any old API patterns (Engine_use, TUI_Window_use, CEL_Build, CEL_Run)

The example should open a blank terminal window at 30fps and exit cleanly on 'q' or Ctrl+C.
