**src/window/tui_window.c** -- Major refactor. Keep all ncurses implementation logic, replace the wrapping:

1. **Remove** all of these:
   - `Engine_WindowState_t Engine_WindowState` global and `Engine_WindowState_register()`
   - `CELS_BackendDesc tui_backend_desc` struct and `cels_backend_register()` call
   - `TUI_Window_use()` function
   - `tui_window_get_standard_state()` and `CELS_WindowState g_window_state`
   - `g_cel_window_entity` and all `CEL_Window`/`CEL_WindowID` references
   - `cels_state_notify_change`, `cels_component_notify_change`, `cels_window_set` calls
   - `#include <cels/backend.h>`

2. **Keep** (the battle-tested ncurses internals):
   - `tui_hook_startup()` -- but extract it into a new static function `ncurses_terminal_init(NCurses_WindowConfig* config)` that takes the component data as input instead of reading from `g_tui_config`. It should: set signal handler, atexit, setlocale, initscr, cbreak, noecho, keypad, curs_set, nodelay, ESCDELAY, color init, and store fps for timing.
   - `tui_hook_shutdown()` -- rename to `ncurses_terminal_shutdown()`. Just does endwin/cleanup. Remove Engine_WindowState and g_window_state references.
   - `cleanup_endwin()` atexit handler -- keep as-is
   - `tui_sigint_handler()` -- keep, but set g_running = 0 AND later the observer will set NCurses_WindowState.running = false
   - Frame timing statics (`g_frame_start`, `g_prev_frame_start`, `g_delta_time`) -- keep for now
   - `tui_hook_frame_begin()` -- keep the timing logic, but instead of updating `Engine_WindowState` and `CEL_Window`, update the NCurses_WindowState component on the window entity via `ecs_set_id`. On resize (COLS/LINES changed), update NCurses_WindowState.width/height on the entity. This triggers recomposition for anything using `cel_watch(entity, NCurses_WindowState)`.
   - `tui_hook_frame_end()` -- keep FPS throttle logic

3. **Add new** -- Observer registration functions:

   `ncurses_register_window_observer()`:
   - Creates a flecs observer watching `EcsOnSet` events on `NCurses_WindowConfig_id`
   - Callback: `on_window_config_set(ecs_iter_t* it)`
     - Check `g_window_entity != 0` -- if already set, log warning "NCurses: only one window entity allowed, ignoring" and return
     - Get `NCurses_WindowConfig* configs = ecs_field(it, NCurses_WindowConfig, 0);`
     - Store `g_window_entity = it->entities[i]`
     - Call `ncurses_terminal_init(&configs[i])` (the extracted startup logic)
     - Attach NCurses_WindowState to the entity:
       ```c
       NCurses_WindowState state = {
           .width = COLS,
           .height = LINES,
           .running = true,
           .actual_fps = (float)(configs[i].fps > 0 ? configs[i].fps : 60),
           .delta_time = 1.0f / (float)(configs[i].fps > 0 ? configs[i].fps : 60)
       };
       ecs_set_id(world, it->entities[i], NCurses_WindowState_id,
                  sizeof(NCurses_WindowState), &state);
       ```
   - Observer pattern from RESEARCH.md Pattern 2 (see ecs_observer_desc_t usage)
   - Must call `cels_ensure_component` for NCurses_WindowConfig_id before using it in the observer query (it may already be ensured by module init, but belt-and-suspenders)

   `ncurses_register_window_remove_observer()`:
   - Creates a flecs observer watching `EcsOnRemove` events on `NCurses_WindowConfig_id`
   - Callback: calls `ncurses_terminal_shutdown()`, resets `g_window_entity = 0`

4. **Add** a frame-tick function that the frame system can call to update window state:
   `ncurses_window_frame_update()` -- checks COLS/LINES for resize, updates NCurses_WindowState on g_window_entity if changed. Also updates delta_time each frame. This replaces the old `tui_hook_frame_begin` window-update portion.

5. **Includes**: Add `#include "cels-ncurses/tui_ncurses.h"` (for NCurses_WindowConfig, NCurses_WindowState types and their _id variables). Keep `#include <flecs.h>`, `#include <ncurses.h>`, timing/signal headers.

6. **g_running and quit integration**: When `tui_sigint_handler` fires (SIGINT), set `g_running = 0`. The frame update function checks `g_running` and if false, sets `NCurses_WindowState.running = false` on the entity AND calls `cels_request_quit()`. This keeps `cels_running()` working naturally.

7. **Keep `tui_window_get_running_ptr()`** for now -- the input system uses it to set `*g_tui_running_ptr = 0` on 'q'. Plan 02 Task 2 will update tui_input.c to call `cels_request_quit()` instead, but the running_ptr bridge keeps things working during the transition.

**include/cels-ncurses/tui_window.h** -- Strip down to minimal:
- Remove `#include <cels/backend.h>` (no more backend descriptor)
- Remove `Engine_WindowState_t` struct and extern (replaced by NCurses_WindowState)
- Remove `Engine_WindowState_id`, `Engine_WindowState_register()`
- Remove `TUI_Window` config struct and `TUI_Window_use()` (replaced by entity-component)
- Remove `tui_window_get_standard_state()` (no more CELS_WindowState)
- Remove all backward compatibility typedefs/macros
- Keep `WindowState` enum (NONE, READY, CLOSING, CLOSED) -- still useful internally
- Keep `tui_window_get_running_ptr()` -- bridge for input system
- Add `extern void ncurses_window_frame_update(void);` -- called by frame system
- This header is now an internal-facing header (not part of the public API surface; the public types are in tui_ncurses.h). It's still needed by tui_frame.c and tui_input.c for internal functions.

**src/frame/tui_frame.c** -- Add the `ncurses_register_frame_systems()` function:
- This function replaces the weak stub in ncurses_module.c
- It should call the existing `tui_frame_register_systems()` (which already creates ECS systems at EcsPreStore and EcsPostFrame)
- Also call `tui_frame_init()` (which the old Engine_init body used to call)
- Add `ncurses_window_frame_update()` call at the start of the frame_begin system callback (so window state updates before frame processing). Import via `extern void ncurses_window_frame_update(void);` or include tui_window.h.
- The function signature `void ncurses_register_frame_systems(void)` must match the declaration in `tui_internal.h`
- IMPORTANT: Place `ncurses_register_frame_systems()` inside the existing `#ifdef CELS_HAS_ECS` block, alongside `tui_frame_register_systems()`. The standalone `draw_test` target does NOT define `CELS_HAS_ECS`, so any ECS-dependent code outside this guard will break the draw_test build. The new function uses `tui_frame_register_systems()` and `tui_frame_init()` which are themselves ECS-dependent, so it must be guarded.

Implementation:
```c
#ifdef CELS_HAS_ECS
/* ... existing tui_frame_register_systems() ... */

void ncurses_register_frame_systems(void) {
    tui_frame_init();
    tui_frame_register_systems();
}
#endif /* CELS_HAS_ECS */
```

And in the existing `tui_frame_begin_system` callback (the ECS system that runs at EcsPreStore), add `ncurses_window_frame_update();` as the first call -- this updates window dimensions and delta_time on the entity each frame before the frame pipeline runs.

**src/input/tui_input.c** -- Add the `ncurses_register_input_system()` function:
- This function replaces the weak stub in ncurses_module.c
- The existing `TUI_Input_use()` function does system registration. Extract the system registration code into `ncurses_register_input_system()` and have it called by the module init instead of by `TUI_Input_use()`.
- Keep `TUI_Input_use()` as a no-op or remove it (the old provider pattern is dead). If removing causes issues with tui_engine.c (which Plan 03 deletes), keep it as a no-op stub for now.
- The input system callback that currently calls `*g_tui_running_ptr = 0` on 'q' key should also call `cels_request_quit()` so that `cels_running()` returns false. Import via `#include <cels/cels.h>` which should already be included.
- The function signature `void ncurses_register_input_system(void)` must match `tui_internal.h`
- Keep `g_input_state` global for now -- it will become an NCurses_InputState component on a separate entity in Phase 3. The global remains accessible for backward compat during transition.
