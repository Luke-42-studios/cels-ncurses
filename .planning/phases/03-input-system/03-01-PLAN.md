<objective>
Define the NCurses_InputState component and rewrite tui_input.c to write input state onto the window entity each frame instead of using a global variable.

Purpose: Transforms input from a global-state pattern (g_input_state + getters) to an ECS component pattern where input state lives on the window entity, matching the established NCurses_WindowState pattern from Phase 1.2. This satisfies INP-01 (system reads input at OnLoad) and INP-02 (developer reads component after input phase).

Output: NCurses_InputState component defined in public header, registered in module composition, written each frame by the input system via cels_entity_set_component + cels_component_notify_change.
</objective>

    1. After the NCurses_WindowState CEL_Component block (around line 98), add:

    ```c
    /*
     * Per-frame input state. NCurses input system fills this each frame at OnLoad.
     * Developer reads via cel_watch(entity, NCurses_InputState) or
     * cels_entity_get_component(entity, NCurses_InputState_id).
     *
     * Per-frame fields reset each frame. Persistent fields (mouse position,
     * held buttons) carry across frames.
     */
    CEL_Component(NCurses_InputState) {
        /* Keyboard: directional input (-1.0/0.0/1.0 for x/y) */
        float axis_x;
        float axis_y;

        /* Keyboard: action buttons (per-frame) */
        bool accept;        /* Enter/Return */
        bool cancel;        /* Escape */

        /* Keyboard: navigation (per-frame) */
        bool tab;
        bool shift_tab;
        bool home;
        bool end;
        bool page_up;
        bool page_down;
        bool backspace;
        bool delete_key;

        /* Keyboard: number keys (per-frame) */
        int  number;
        bool has_number;

        /* Keyboard: function keys (per-frame) */
        int  function_key;
        bool has_function;

        /* Keyboard: raw key passthrough (per-frame) */
        int  raw_key;
        bool has_raw_key;

        /* Mouse: position (persistent across frames) */
        int mouse_x;
        int mouse_y;

        /* Mouse: button event (per-frame) */
        int  mouse_button;   /* 0=none, 1=left, 2=middle, 3=right */
        bool mouse_pressed;
        bool mouse_released;

        /* Mouse: held state (persistent across frames) */
        bool mouse_left_held;
        bool mouse_middle_held;
        bool mouse_right_held;
    };
    ```

    **src/input/tui_input.c** -- Rewrite to use entity-component pattern instead of global state. The input reading switch/case logic is PRESERVED nearly verbatim. Only the storage and access pattern changes.

    Full rewrite of the file. Key changes:

    1. **Includes:** Replace `#include "cels-ncurses/tui_input.h"` with `#include "cels-ncurses/tui_ncurses.h"` and `#include "cels-ncurses/tui_internal.h"`.     2. **Remove globals:** Delete `TUI_InputState g_input_state`, `g_tui_running_ptr`, and `g_mouse_initialized`. Keep `g_quit_guard_fn` as internal function pointer (research recommendation: keep as internal API for now).

    3. **Keep constants:** CELS_KEY_CTRL_UP/DOWN/LEFT/RIGHT, CELS_KEY_SHIFT_LEFT/RIGHT stay as-is.

    4. **Keep tui_pause():** Unchanged.

    5. **Rewrite tui_read_input_ncurses():**
       - Get window entity via `ncurses_window_get_entity()`. If 0, return.
       - Read previous state via `cels_entity_get_component(win, NCurses_InputState_id)` for persistent fields.
       - Build fresh `NCurses_InputState state = {0}` on the stack.
       - Copy persistent fields from prev: mouse_x, mouse_y, mouse_left_held, mouse_middle_held, mouse_right_held.
       - Call `wgetch(stdscr)`. If ERR, still write the zeroed per-frame state to the entity (so state is always fresh).
       - Switch/case logic preserved but writes to local `state` struct instead of `g_input_state`. Field name mapping:
         - `axis_left[0]` -> `state.axis_x`
         - `axis_left[1]` -> `state.axis_y`
         - `button_accept` -> `state.accept`
         - `button_cancel` -> `state.cancel`
         - `key_tab` -> `state.tab`
         - `key_shift_tab` -> `state.shift_tab`
         - `key_home` -> `state.home`
         - `key_end` -> `state.end`
         - `key_page_up` -> `state.page_up`
         - `key_page_down` -> `state.page_down`
         - `key_backspace` -> `state.backspace`
         - `key_delete` -> `state.delete_key`
         - `key_number` -> `state.number`
         - `key_function` -> `state.function_key`
         - `mouse_button` enum values: Use plain integers (0=none, 1=left, 2=middle, 3=right) instead of TUI_MOUSE_* enum (that enum is being removed).
       - Quit case ('q'/'Q'): Remove `if (g_tui_running_ptr) *g_tui_running_ptr = 0;` -- keep only `cels_request_quit(); return;`. Quit guard logic preserved.
       - KEY_RESIZE: Keep as-is (calls tui_layer_resize_all, tui_frame_invalidate_all, clearok).
       - KEY_MOUSE drain loop: Preserved exactly, writing to local `state` instead of global.
       - After switch/case (before return), write to entity:
         ```c
         cels_entity_set_component(win, NCurses_InputState_id,
                                    &state, sizeof(NCurses_InputState));
         cels_component_notify_change(NCurses_InputState_id);
         ```
       - IMPORTANT: The entity write must happen after the switch block, including when ch == ERR (write the zeroed-but-persistent-copied state). The only cases that return early without writing are: quit (cels_request_quit), KEY_RESIZE, and tui_pause. For KEY_MOUSE, do NOT return early -- fall through to the entity write after the drain loop finishes (remove the `return;` after the mouse drain; let it fall through to the write).

    6. **Remove getter functions:** Delete `tui_input_get_state()` and `tui_input_get_mouse_position()`.

    7. **CEL_System unchanged:** `CEL_System(NCurses_InputSystem, .phase = OnLoad)` stays exactly as-is.

    8. **ncurses_register_input_system():** Remove `g_tui_running_ptr = tui_window_get_running_ptr();`. Remove `g_input_state.mouse_x = -1; g_input_state.mouse_y = -1;` (initial mouse pos is now in composition cel_has). Keep define_key calls, mousemask/mouseinterval calls, g_mouse_initialized flag (still useful to gate mousemask to one call), and NCurses_InputSystem_register(). Actually, remove g_mouse_initialized since it was only checked once -- the registration function is only called once.

    9. **Keep quit guard API:** `tui_input_set_quit_guard()` stays as-is with its `g_quit_guard_fn` static. Keep its declaration somewhere accessible -- add it to tui_internal.h or keep it in the stripped tui_input.h.

     *
     * Internal-only declarations for the input subsystem.
     * Public input type (NCurses_InputState) is in tui_ncurses.h.

    Remove: TUI_MouseButton enum, TUI_InputState typedef, extern g_input_state, tui_input_get_state(), tui_input_get_mouse_position(), CELS_Input typedef, cels_input_get() shim.
