  <files>examples/draw_test.c</files>
  <action>
**In main(), after the start_color() call (line ~1005), add:**
```c
    start_color();

    /* Initialize three-tier color system (auto-detect) */
    tui_color_init(-1);
```

This must come AFTER `start_color()` and BEFORE `init_styles()` because `tui_color_init()` needs ncurses color system active, and `init_styles()` calls `tui_color_rgb()` which depends on the detected mode.

**In the draw_hud() helper function**, add the color mode to the FPS display line. Change:
```c
snprintf(buf, sizeof(buf), "FPS: %.0f  Frame: %lu  dt: %.3f",
         g_frame_state.fps, (unsigned long)g_frame_state.frame_count,
         g_frame_state.delta_time);
```
to:
```c
snprintf(buf, sizeof(buf), "FPS: %.0f  Frame: %lu  color: %s",
         g_frame_state.fps, (unsigned long)g_frame_state.frame_count,
         tui_color_mode_name(tui_color_get_mode()));
```

This shows the active color mode (256/palette/direct) in every scene's HUD, giving developers immediate visual feedback.

**In draw_menu()**, add a similar color mode indicator. After the existing FPS display at the bottom of draw_menu():
```c
char fps_buf[32];
snprintf(fps_buf, sizeof(fps_buf), "FPS: %.0f", g_frame_state.fps);
tui_draw_text(ctx, cols - (int)strlen(fps_buf) - 1, 0, fps_buf, s_hud);
```
Add below it:
```c
char mode_buf[32];
snprintf(mode_buf, sizeof(mode_buf), "Color: %s", tui_color_mode_name(tui_color_get_mode()));
```
