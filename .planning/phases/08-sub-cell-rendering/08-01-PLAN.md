    ```
     * Sub-Cell Drawing — Half-Block Mode
     * ============================================================================
     *
     * Half-block mode: 1x2 virtual pixels per terminal cell.
     * pixel_x maps 1:1 to cell_x. pixel_y / 2 = cell_y, pixel_y % 2 = sub (0=top, 1=bottom).
     * Uses U+2584 (lower half block) as canonical character: fg=bottom, bg=top.
     * Drawing one half preserves the other half's existing color.
     * Requires layer-backed DrawContext (ctx->subcell_buf != NULL).
     */
    ```

    Declare:
    - `extern void tui_draw_halfblock_plot(TUI_DrawContext* ctx, int px, int py, TUI_Style style);`

    **Implement in `src/graphics/tui_draw.c`:**

    Add `#include "cels-ncurses/tui_subcell.h"` to the includes.

    **Static helper -- ensure buffer exists (lazy alloc):**
    ```c
    static TUI_SubCellBuffer* subcell_ensure_buffer(TUI_DrawContext* ctx) {
        if (!ctx->subcell_buf) return NULL;  /* non-layer context */
        if (!*ctx->subcell_buf) {
            *ctx->subcell_buf = tui_subcell_buffer_create(ctx->width, ctx->height);
        }
        return *ctx->subcell_buf;
    }
    ```

    **Static helper -- write one half-block cell to ncurses:**
    Follow the exact pattern from research (subcell_write_halfblock). Key logic:
    - Both halves drawn: use U+2584 (0x2584), fg = bottom color, bg = top color
    - Only top drawn: use U+2580 (0x2580), fg = top color, bg = TUI_COLOR_DEFAULT
    - Only bottom drawn: use U+2584 (0x2584), fg = bottom color, bg = TUI_COLOR_DEFAULT
    - Neither drawn: return (nothing to write)

    Construct cchar_t via setcchar() with A_NORMAL, pair 0, NULL opts. Then call tui_style_apply() with the constructed style, then mvwadd_wch(). This follows the existing pattern in tui_draw.c (see init_rounded_corners and tui_draw_border_rect).

    **tui_draw_halfblock_plot(ctx, px, py, style):**
    1. Get buffer via subcell_ensure_buffer. Return if NULL.
    2. Convert: cell_x = px, cell_y = py / 2, sub_y = py % 2 (0=top, 1=bottom)
    3. Bounds check: cell_x and cell_y must be within buffer dimensions
    4. Scissor clip: `if (!tui_cell_rect_contains(ctx->clip, cell_x, cell_y)) return;`
    5. Get cell: `&buf->cells[cell_y * buf->width + cell_x]`
    6. Mode mixing -- last mode wins: if cell->mode != TUI_SUBCELL_HALFBLOCK, memset cell to 0, set mode
    7. Set the appropriate half: if sub_y == 0: top = style.fg, has_top = true; else: bottom = style.fg, has_bottom = true
    8. Call subcell_write_halfblock to write-through to ncurses

    **tui_draw_halfblock_fill_rect(ctx, px, py, pw, ph, style):**
    1. Get buffer via subcell_ensure_buffer. Return if NULL.
    2. Loop over pixel_y from py to py+ph-1, pixel_x from px to px+pw-1:
       - Convert: cell_x = pixel_x, cell_y = pixel_y / 2, sub_y = pixel_y % 2
       - Bounds check against buffer dimensions
       - Scissor clip at cell level: `if (!tui_cell_rect_contains(ctx->clip, cell_x, cell_y)) continue;`
       - Mode mixing: last mode wins (reset cell if not HALFBLOCK)
       - Set the half color
    3. After the inner pixel loop completes for each unique cell_y, call subcell_write_halfblock for each touched cell.

    OPTIMIZATION: For fill_rect, avoid calling subcell_write_halfblock per-pixel. Instead, track which cells were modified and write them once after the loop. A simpler approach: iterate in cell coordinates, determine which halves the pixel rect covers for each cell, update both at once, then write. The pixel rect in cell space is:
    - cell_y_min = py / 2, cell_y_max = (py + ph - 1) / 2
    - cell_x_min = px, cell_x_max = px + pw - 1
    - For each cell, determine if top half (py <= cell_y*2 < py+ph) and/or bottom half (py <= cell_y*2+1 < py+ph) are covered.

    This approach iterates cells once, not pixels, and is cleaner for fill_rect.

    IMPORTANT: style.fg is the pixel color. Both the top and bottom halves of a filled rect use the same color (style.fg).
