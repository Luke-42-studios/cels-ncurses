    **Add declarations to `include/cels-ncurses/tui_draw.h`:**

    Add a new section after the Half-Block section:

    ```
    /* ============================================================================
     * Sub-Cell Drawing — Quadrant Mode
     * ============================================================================
     *
     * Quadrant mode: 2x2 virtual pixels per terminal cell.
     * pixel_x / 2 = cell_x, pixel_y / 2 = cell_y.
     * (pixel_x % 2, pixel_y % 2) selects one of four quadrants: UL(0,0) UR(1,0) LL(0,1) LR(1,1).
     * Uses 16-entry lookup table mapping 4-bit mask to Unicode codepoints
     * (U+2596-U+259F range plus block element fallbacks for space, half blocks, full block).
     * Two-color constraint: each cell has fg (filled quadrants) and bg (empty quadrants).
     * Requires layer-backed DrawContext (ctx->subcell_buf != NULL).
     */
    ```

    Declare:
    - `extern void tui_draw_quadrant_plot(TUI_DrawContext* ctx, int px, int py, TUI_Style style);`
    - `extern void tui_draw_quadrant_fill_rect(TUI_DrawContext* ctx, int px, int py, int pw, int ph, TUI_Style style);`

    **Implement in `src/graphics/tui_draw.c`:**

    **Static lookup table -- all 16 quadrant combinations:**
    ```c
    static const wchar_t QUADRANT_CHARS[16] = {
        L' ',    /* 0b0000 = empty */
        0x2598,  /* 0b0001 = UL */
        0x259D,  /* 0b0010 = UR */
        0x2580,  /* 0b0011 = UL+UR (upper half block) */
        0x2596,  /* 0b0100 = LL */
        0x258C,  /* 0b0101 = UL+LL (left half block) */
        0x259E,  /* 0b0110 = UR+LL */
        0x259B,  /* 0b0111 = UL+UR+LL */
        0x2597,  /* 0b1000 = LR */
        0x259A,  /* 0b1001 = UL+LR */
        0x2590,  /* 0b1010 = UR+LR (right half block) */
        0x259C,  /* 0b1011 = UL+UR+LR */
        0x2584,  /* 0b1100 = LL+LR (lower half block) */
        0x2599,  /* 0b1101 = UL+LL+LR */
        0x259F,  /* 0b1110 = UR+LL+LR */
        0x2588,  /* 0b1111 = full block */
    };
    ```

    **Static helper -- convert (sub_x, sub_y) to quadrant bit:**
    bit0=UL(0,0), bit1=UR(1,0), bit2=LL(0,1), bit3=LR(1,1)
    Formula: `(uint8_t)(1 << (sy * 2 + sx))`

    **Static helper -- write one quadrant cell to ncurses (subcell_write_quadrant):**
    1. Get cell->quadrant.mask. If mask == 0 (all empty), return.
    2. If mask == 0x0F (all filled), use full block U+2588 with fg = cell->quadrant.fg, bg = cell->quadrant.fg (same color).
    3. Otherwise: look up QUADRANT_CHARS[mask]. Style: fg = cell->quadrant.fg (filled), bg = cell->quadrant.bg (empty/background).
    4. Construct cchar_t via setcchar, call tui_style_apply, then mvwadd_wch. Same pattern as halfblock write-through.

    **tui_draw_quadrant_plot(ctx, px, py, style):**
    1. Get buffer via subcell_ensure_buffer. Return if NULL.
    2. Convert: cell_x = px / 2, cell_y = py / 2, sub_x = px % 2, sub_y = py % 2
    3. Bounds check cell_x/cell_y against buffer dimensions
    4. Scissor clip: `if (!tui_cell_rect_contains(ctx->clip, cell_x, cell_y)) return;`
    5. Get cell. Mode mixing: if mode != QUADRANT, memset cell to 0, set mode = QUADRANT, set bg = TUI_COLOR_DEFAULT
    6. Set the quadrant bit: `cell->quadrant.mask |= quadrant_bit(sub_x, sub_y);`
    7. Two-color handling: set `cell->quadrant.fg = style.fg;` -- the fg color is the "filled" color for all filled quadrants. If the cell had a different fg before, the new one wins (last-write-wins for fg color). The bg color stays as whatever it was (defaults to TUI_COLOR_DEFAULT).
    8. Call subcell_write_quadrant to write-through.

    **tui_draw_quadrant_fill_rect(ctx, px, py, pw, ph, style):**
    1. Get buffer. Return if NULL.
    2. Iterate in cell coordinates:
       - cell_y_min = py / 2, cell_y_max = (py + ph - 1) / 2
       - cell_x_min = px / 2, cell_x_max = (px + pw - 1) / 2
    3. For each cell:
       - Bounds check, scissor clip
       - Mode mixing: reset if not QUADRANT
       - Determine which quadrants the pixel rect covers in this cell:
         - For each sub position (sx, sy) in {0,1}x{0,1}: pixel coordinate would be cell_x*2+sx, cell_y*2+sy. If that pixel is within [px, px+pw) x [py, py+ph), set the quadrant bit.
       - Set fg = style.fg
       - Write-through

    ```
    /* ============================================================================
     * Sub-Cell Drawing — Braille Mode
     * ============================================================================
     *
     * Braille mode: 2x4 virtual pixels per terminal cell.
     * pixel_x / 2 = cell_x, pixel_y / 4 = cell_y.
     * (pixel_x % 2, pixel_y % 4) selects one of 8 dot positions.
     * Braille bit ordering is non-sequential: use BRAILLE_DOT_BIT[sub_x][sub_y] lookup.
     * Codepoint = U+2800 + dots_bitmask (direct binary encoding by Unicode design).
     * Read-modify-write: plot sets a dot (OR), unplot clears a dot (AND NOT).
     * Requires layer-backed DrawContext (ctx->subcell_buf != NULL).
     */
    ```

    Declare:
    - `extern void tui_draw_braille_plot(TUI_DrawContext* ctx, int px, int py, TUI_Style style);`
    - `extern void tui_draw_braille_unplot(TUI_DrawContext* ctx, int px, int py);`
    - `extern void tui_draw_braille_fill_rect(TUI_DrawContext* ctx, int px, int py, int pw, int ph, TUI_Style style);`

    Also add the resolution query section:

    ```
    /* ============================================================================
     * Sub-Cell Resolution Query
     * ============================================================================
     *
     * Returns the virtual pixel dimensions of the DrawContext for a given sub-cell mode.
     * Renderers like Clay use this to calculate their logical viewport size.
     */
    ```

    Declare:
    - `extern void tui_draw_subcell_resolution(TUI_DrawContext* ctx, TUI_SubCellMode mode, int* width, int* height);`

    **Implement braille in `src/graphics/tui_draw.c`:**

    **Static lookup table -- braille dot-to-bit mapping:**
    ```c
    static const uint8_t BRAILLE_DOT_BIT[2][4] = {
        /* col 0 (left)  */ { 0x01, 0x02, 0x04, 0x40 },  /* bits 0, 1, 2, 6 */
        /* col 1 (right) */ { 0x08, 0x10, 0x20, 0x80 },  /* bits 3, 4, 5, 7 */
    };
    ```

    NOTE: bits 6 and 7 are NOT sequential with bits 0-5. This is the Unicode braille standard. The lookup table handles this correctly.

    **Static helper -- write one braille cell to ncurses (subcell_write_braille):**
    1. Get cell->braille.dots. If dots == 0, write a space character (or return -- either is fine, writing space clears previous content).
    2. Compute codepoint: `wchar_t wc[2] = { 0x2800 + cell->braille.dots, L'\0' };`
    3. Construct cchar_t via setcchar. Style: fg = cell->braille.fg, bg = TUI_COLOR_DEFAULT, attrs = TUI_ATTR_NORMAL.
    4. Call tui_style_apply, then mvwadd_wch. Same pattern as other modes.

    **tui_draw_braille_plot(ctx, px, py, style):**
    1. Get buffer via subcell_ensure_buffer. Return if NULL.
    2. Convert: cell_x = px / 2, cell_y = py / 4, sub_x = px % 2, sub_y = py % 4
    3. Bounds check, scissor clip (same as other modes)
    4. Get cell. Mode mixing: if mode != BRAILLE, memset cell to 0, set mode = BRAILLE
    5. Set dot: `cell->braille.dots |= BRAILLE_DOT_BIT[sub_x][sub_y];`
    6. Set color: `cell->braille.fg = style.fg;`
    7. Call subcell_write_braille to write-through

    **tui_draw_braille_unplot(ctx, px, py):**
    1. Get buffer. Return if NULL (no buffer = nothing to unplot).
    2. Convert coordinates as above.
    3. Bounds check, scissor clip.
    4. Get cell. If mode != BRAILLE, return (nothing to unplot).
    5. Clear dot: `cell->braille.dots &= ~BRAILLE_DOT_BIT[sub_x][sub_y];`
    6. Call subcell_write_braille to write-through (will write space if dots == 0)
    Note: unplot does NOT allocate a buffer -- it only operates on existing buffer. Check `*ctx->subcell_buf == NULL` and return early if no buffer exists.

    **tui_draw_braille_fill_rect(ctx, px, py, pw, ph, style):**
    1. Get buffer. Return if NULL.
    2. Iterate in cell coordinates:
       - cell_y_min = py / 4, cell_y_max = (py + ph - 1) / 4
       - cell_x_min = px / 2, cell_x_max = (px + pw - 1) / 2
    3. For each cell:
       - Bounds check, scissor clip
       - Mode mixing: reset if not BRAILLE
       - For each dot position (sub_x in 0..1, sub_y in 0..3):
         - pixel coordinate = cell_x*2+sub_x, cell_y*4+sub_y
         - If within [px, px+pw) x [py, py+ph): set the dot bit via OR
       - Set fg = style.fg
       - Write-through

    **Implement resolution query:**

    `tui_draw_subcell_resolution(ctx, mode, width, height)`:
    ```c
    switch (mode) {
        case TUI_SUBCELL_HALFBLOCK:
            *width = ctx->width;
            *height = ctx->height * 2;
            break;
        case TUI_SUBCELL_QUADRANT:
            *width = ctx->width * 2;
            *height = ctx->height * 2;
            break;
        case TUI_SUBCELL_BRAILLE:
            *width = ctx->width * 2;
            *height = ctx->height * 4;
            break;
        default:
            *width = ctx->width;
            *height = ctx->height;
            break;
    }
    ```

    Note: TUI_SUBCELL_NONE falls to default and returns cell dimensions (1:1).
