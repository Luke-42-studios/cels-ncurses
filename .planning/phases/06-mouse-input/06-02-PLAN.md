Update `examples/space_invaders/game.c` to use the new input API.

**Search and replace the following patterns:**

1. Replace `CELS_Input` type references:
   - `static CELS_Input g_prev_input = {0};` becomes `static TUI_InputState g_prev_input = {0};`

2. Replace `cels_input_get()` calls:
   - `const CELS_Input* input = cels_input_get(ctx);` becomes `const TUI_InputState* input = tui_input_get_state();`
   - Remove the `CELS_Context* ctx = cels_get_context();` line if it was only used for `cels_input_get(ctx)` (check if ctx is used for anything else in each function -- it likely is used for `cels_get_world(ctx)` too, so only remove if truly unused).

3. Replace `memcpy` of input state:
   - `memcpy((void*)&g_prev_input, input, sizeof(CELS_Input));` becomes `memcpy((void*)&g_prev_input, input, sizeof(TUI_InputState));`

4. Do NOT remove `#include <cels/cels.h>` -- the game uses ECS functions from cels core.

5. Verify that the field names used by space_invaders match the new struct (they should, since TUI_InputState preserves all field names from the old CELS_Input): `axis_left`, `button_accept`, `raw_key`, `has_raw_key`, etc.

**IMPORTANT:** Do not change any game logic. Only change the type names and getter function calls. The field names are identical between old CELS_Input and new TUI_InputState.
