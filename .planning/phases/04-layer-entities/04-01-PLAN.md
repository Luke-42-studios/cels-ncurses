2. **Registry helper functions:**

3. **Z-order synchronization function:**

```c
static void layer_sync_z_order(void) {
    if (!g_z_order_dirty || g_layer_entry_count == 0) return;
    g_z_order_dirty = false;

    /* Insertion sort by z_order ascending (max 32 entries) */
    for (int i = 1; i < g_layer_entry_count; i++) {
        LayerEntry key = g_layer_entries[i];
        int j = i - 1;
        while (j >= 0 && g_layer_entries[j].z_order > key.z_order) {
            g_layer_entries[j + 1] = g_layer_entries[j];
            j--;
        }
        g_layer_entries[j + 1] = key;
    }

    /* Rebuild panel stack: top_panel each in ascending z_order */
    for (int i = 0; i < g_layer_entry_count; i++) {
        if (g_layer_entries[i].panel) {
            top_panel(g_layer_entries[i].panel);
        }
    }
}
```

4. **CEL_Lifecycle and CEL_Observe for TUI_LayerLC:**

on_create observer:
- Read TUI_Layer component from entity via cels_entity_get_component
- Determine dimensions: if width=0 or height=0, use COLS/LINES (fullscreen sentinel)
- Create ncurses WINDOW via newwin(h, w, y, x)
- Create PANEL via new_panel(win)
- set_panel_userptr(panel, (void*)(uintptr_t)entity) for reverse lookup
- Apply initial visibility: if !config->visible, call hide_panel
- Build TUI_DrawContext_Component with ctx from tui_draw_context_create, dirty=false, panel, win, subcell_buf=NULL
- Set the subcell_buf pointer: ctx.subcell_buf = &dc.subcell_buf (BUT see Pitfall 4 -- after cels_entity_set_component copies the struct, this pointer is dangling. Solution: set subcell_buf pointer to NULL initially, and the draw functions that need it will handle it. OR store subcell_buf in the registry instead of the component. Follow the simpler approach: just set ctx.subcell_buf = NULL. Sub-cell buffer allocation is lazy and happens in draw functions anyway.)
- cels_entity_set_component(entity, TUI_DrawContext_Component_id, &dc, sizeof(dc))
- cels_component_notify_change(TUI_DrawContext_Component_id)
- layer_registry_add(entity, config->z_order, panel)

on_destroy observer:
- Read TUI_DrawContext_Component from entity
- Clean up subcell_buf if allocated (tui_subcell_buffer_destroy)
- del_panel(panel), delwin(win)
- layer_registry_remove(entity)

5. **Per-frame z-order sync system:**

```c
CEL_System(TUI_LayerSyncSystem, .phase = PreRender) {
    cel_run {
        /* Check each registry entry against current TUI_Layer component */
        for (int i = 0; i < g_layer_entry_count; i++) {
            const TUI_Layer* layer = cels_entity_get_component(
                g_layer_entries[i].entity, TUI_Layer_id);
            if (!layer) continue;

            /* Detect z_order change */
            if (layer->z_order != g_layer_entries[i].z_order) {
                g_layer_entries[i].z_order = layer->z_order;
                g_z_order_dirty = true;
            }

            /* Detect visibility change */
            /* Read current DrawContext to get panel pointer */
            const TUI_DrawContext_Component* dc = cels_entity_get_component(
                g_layer_entries[i].entity, TUI_DrawContext_Component_id);
            if (dc && dc->panel) {
                if (layer->visible && panel_hidden(dc->panel)) {
                    show_panel(dc->panel);
                } else if (!layer->visible && !panel_hidden(dc->panel)) {
                    hide_panel(dc->panel);
                }
            }
        }

        layer_sync_z_order();
    }
}
```

NOTE: The visibility check uses panel_hidden() from ncurses panels API to compare against the TUI_Layer.visible field. This avoids needing a cached visible flag in the registry.
