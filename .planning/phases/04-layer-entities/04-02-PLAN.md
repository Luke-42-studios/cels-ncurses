**Update `src/frame/tui_frame.c`:**

The frame pipeline needs to clear dirty entity-managed layers at frame_begin, alongside the existing g_layers[] clearing. The entity layer registry is in tui_layer_entity.c which is a separate TU. Since this is an INTERFACE library, we need to access the entity layer data.

Two approaches:
- (A) Expose the entity layer registry via extern declarations and iterate it in tui_frame.c
- (B) Add a function in tui_layer_entity.c that clears dirty entity layers, and call it from tui_frame.c

Use approach (B) -- cleaner encapsulation. Add a function `ncurses_layer_entity_clear_dirty()` in tui_layer_entity.c that iterates g_layer_entries, checks dirty flags on each entity's TUI_DrawContext_Component, and clears them with werase + style reset + subcell buffer clear (same pattern as the existing g_layers[] loop in tui_frame_begin).

In tui_layer_entity.c, add:
```c
void ncurses_layer_entity_clear_dirty(void) {
    for (int i = 0; i < g_layer_entry_count; i++) {
        TUI_DrawContext_Component* dc = (TUI_DrawContext_Component*)
            cels_entity_get_component(g_layer_entries[i].entity,
                                       TUI_DrawContext_Component_id);
        if (!dc) continue;
        if (dc->dirty && dc->win) {
            werase(dc->win);
            wattr_set(dc->win, A_NORMAL, 0, NULL);
            if (dc->subcell_buf) {
                tui_subcell_buffer_clear(dc->subcell_buf);
            }
            dc->dirty = false;
        }
    }
}
```

WAIT -- cels_entity_get_component returns `const void*`. We need to mutate the dirty flag. Options:
- Cast away const (fragile but works since we own the data)
- Use cels_entity_set_component to write back with dirty=false (heavyweight)
- Track dirty in the registry instead of the component

Simplest: track dirty per-entry in the LayerEntry struct in the registry. Add `bool dirty;` to LayerEntry. The draw functions set dirty on the DrawContext component; the clear function reads from the component and resets it. BUT the component is const...

Actually, the simplest approach: the `tui_layer_get_draw_context` function (existing API) sets dirty=true on the TUI_Layer struct. For entity layers, the developer gets TUI_DrawContext via cel_watch. We need a mechanism to mark entity layers dirty when drawn into.

Alternative approach that avoids the const problem entirely: **Track dirty in the registry's LayerEntry.** When the developer draws into a layer, they call tui_draw_* with a DrawContext whose WINDOW* belongs to an entity layer. We can't intercept those calls. Instead, use a simpler heuristic: **mark all visible entity layers as dirty every frame** (conservative approach). This means every visible entity layer gets werase'd at frame_begin, matching the behavior that the developer's composition body re-runs each frame and redraws content.

This is actually the correct behavior for the ECS composition model: the composition body runs every frame, redraws everything, so all visible layers should be cleared. The retained-mode dirty optimization from v1.0 was for the imperative API where layers persisted content between frames. In the ECS model, the composition re-runs, so clearing every visible layer each frame is correct.

Update the clear function to simply clear ALL visible entity layers:
```c
void ncurses_layer_entity_clear_dirty(void) {
    for (int i = 0; i < g_layer_entry_count; i++) {
        const TUI_DrawContext_Component* dc = cels_entity_get_component(
            g_layer_entries[i].entity, TUI_DrawContext_Component_id);
        if (!dc || !dc->win) continue;

        const TUI_Layer* layer = cels_entity_get_component(
            g_layer_entries[i].entity, TUI_Layer_id);
        if (!layer || !layer->visible) continue;

        werase(dc->win);
        wattr_set(dc->win, A_NORMAL, 0, NULL);
        if (dc->subcell_buf) {
            tui_subcell_buffer_clear(dc->subcell_buf);
        }
    }
}
```

Add extern declaration to tui_internal.h (or a new internal header):
```c
extern void ncurses_layer_entity_clear_dirty(void);
```

In tui_frame.c, in the `tui_frame_begin` function, AFTER the existing g_layers[] dirty clearing loop, add (inside the #ifdef CELS_HAS_ECS guard):
```c
#ifdef CELS_HAS_ECS
    ncurses_layer_entity_clear_dirty();
#endif
```

This keeps the old layer clearing for draw_test compatibility and adds entity layer clearing for ECS builds.

2. Inside the window composition body, create TWO TUILayer entities at different z-orders
3. Draw content into each layer using tui_draw_* functions via TUI_DrawContext from cel_watch
4. Handle quit via the existing CEL_System(GameInput) pattern reading NCurses_InputState

Structure:
```c
#include <cels/cels.h>
#include <cels_ncurses.h>
#include <cels_ncurses_draw.h>

CEL_Composition(World) {
    NCursesWindow(.title = "Layer Demo", .fps = 30) {
        const struct NCurses_WindowState* ws = cel_watch(NCurses_WindowState);
        if (!ws) return;

        /* Background layer - fullscreen, z=0 */
        TUILayer(.z_order = 0, .visible = true) {
            const TUI_DrawContext_Component* dc = cel_watch(TUI_DrawContext_Component);
            if (!dc) return;
            TUI_DrawContext ctx = dc->ctx;

            TUI_Style bg_style = {
                .fg = tui_color_rgb(100, 100, 100),
                .bg = tui_color_rgb(20, 20, 40),
                .attrs = TUI_ATTR_NORMAL
            };
            tui_draw_fill_rect(&ctx,
                (TUI_CellRect){0, 0, ctx.width, ctx.height},
                '.', bg_style);
            tui_draw_text(&ctx, 2, 1, "Background Layer (z=0)", bg_style);
        }

        /* Overlay layer - fixed position/size, z=10 */
        TUILayer(.z_order = 10, .visible = true,
                 .x = 5, .y = 3, .width = 40, .height = 12) {
            const TUI_DrawContext_Component* dc = cel_watch(TUI_DrawContext_Component);
            if (!dc) return;
            TUI_DrawContext ctx = dc->ctx;

            TUI_Style box_style = {
                .fg = tui_color_rgb(200, 200, 255),
                .bg = tui_color_rgb(30, 30, 60),
                .attrs = TUI_ATTR_BOLD
            };
            tui_draw_fill_rect(&ctx,
                (TUI_CellRect){0, 0, ctx.width, ctx.height},
                ' ', box_style);
            tui_draw_border_rect(&ctx,
                (TUI_CellRect){0, 0, ctx.width, ctx.height},
                TUI_BORDER_SINGLE, box_style);

            TUI_Style text_style = {
                .fg = tui_color_rgb(255, 255, 100),
                .bg = tui_color_rgb(30, 30, 60),
                .attrs = TUI_ATTR_NORMAL
            };
            tui_draw_text(&ctx, 2, 1, "Overlay Layer (z=10)", text_style);
            tui_draw_text(&ctx, 2, 3, "This layer is on top!", text_style);
            tui_draw_text(&ctx, 2, 5, "Press 'q' to quit", text_style);
        }
    }
}

/* System: reads raw input each frame */
CEL_System(GameInput, .phase = OnUpdate) {
    cel_run {
        const struct NCurses_InputState* input = cel_read(NCurses_InputState);
        if (!input) return;
        for (int i = 0; i < input->key_count; i++) {
            if (input->keys[i] == 'q' || input->keys[i] == 'Q') {
                cels_request_quit();
                return;
            }
        }
    }
}

cels_main() {
    cels_register(NCurses);
    cels_register(GameInput);
    cels_session(World) {
        while (cels_running()) cels_step(0);
    }
}
```
