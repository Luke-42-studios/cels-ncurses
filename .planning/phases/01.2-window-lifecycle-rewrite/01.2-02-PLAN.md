CELS API available (from Plan 01):
- cels_entity_set_component(entity, component_id, data, size) -- set component on any entity
- cels_entity_get_component(entity, component_id) -- get component from any entity (returns const void*)
- cels_component_notify_change(component_id) -- trigger cel_watch reactivity
- cels_lifecycle_bind_entity(lifecycle_id, entity_id) -- bind entity to lifecycle, fires on_create
- CEL_Lifecycle(Name, .active = &Condition) -- file-scope lifecycle definition (static linkage)
- CEL_Observe(Name, event) { body } -- file-scope observer (static linkage, auto-registers via constructor)
- CEL_Condition(Name) { body } -- file-scope condition function (static linkage)

DELIBERATE DEVIATION FROM RESEARCH: The research document recommends placing CEL_Lifecycle,
CEL_Condition, and CEL_Observe in tui_window.c. However, this plan places them in ncurses_module.c
instead. Reason: CEL_Lifecycle creates static variables (NCursesWindowLC_id, etc.) that must be
visible to CEL_Compose for the cels_lifecycle_bind_entity() call. Placing lifecycle, observers,
AND composition in the same translation unit (ncurses_module.c) eliminates cross-TU static
variable visibility issues identified in Research Pitfall 2. The tui_window.c file remains
responsible for terminal init/shutdown implementation and frame update logic.

4. **CEL_Observe on_create** -- terminal initialization only, NO component setting:
   ```c
   CEL_Observe(NCursesWindowLC, on_create) {
       if (ncurses_window_get_entity() != 0) {
           fprintf(stderr, "[NCurses] Warning: only one window entity allowed\n");
           return;
       }
       ncurses_window_set_entity(entity);

       /* Read config from entity using CELS public API */
       const NCurses_WindowConfig* config =
           (const NCurses_WindowConfig*)cels_entity_get_component(entity, NCurses_WindowConfig_id);
       if (!config) {
           fprintf(stderr, "[NCurses] Error: on_create fired but no WindowConfig on entity\n");
           ncurses_window_set_entity(0);
           return;
       }

       ncurses_terminal_init((NCurses_WindowConfig*)config);
   }
   ```
   CRITICAL (SC3): The on_create observer does NOT set NCurses_WindowState. It ONLY calls
   `ncurses_terminal_init()` and `ncurses_window_set_entity(entity)`. The initial WindowState
   is declared via `cel_has` in the CEL_Compose body (see step 7). The very first frame of
   `ncurses_window_frame_update()` will overwrite it with real COLS/LINES values via
   `cels_entity_set_component()`.

   Note: `cels_ensure_component` calls are NOT needed here because CEL_Module init (step 6)
   already registered both component IDs before any entity creation can happen. CEL_Module
   runs during `cels_register(NCurses)` which is before `cels_session()` starts composing
   entities. The module and composition are in the same TU, so the static component ID
   variables are shared.

5. **CEL_Observe on_destroy** -- terminal shutdown:
   ```c
   CEL_Observe(NCursesWindowLC, on_destroy) {
       (void)entity;
       ncurses_terminal_shutdown();
       ncurses_window_set_entity(0);
   }
   ```

6. **CEL_Module(NCurses)** -- simplified init body:
   ```c
   CEL_Module(NCurses) {
       /* Register component types */
       cels_ensure_component(&NCurses_WindowConfig_id, "NCurses_WindowConfig",
                             sizeof(NCurses_WindowConfig), CELS_ALIGNOF(NCurses_WindowConfig));
       cels_ensure_component(&NCurses_WindowState_id, "NCurses_WindowState",
                             sizeof(NCurses_WindowState), CELS_ALIGNOF(NCurses_WindowState));

       /* Register lifecycle (observers auto-register via constructor attribute) */
       NCursesWindowLC_register();

       /* Register systems (frame pipeline) */
       ncurses_register_input_system();
       ncurses_register_frame_systems();
   }
   ```
   Remove calls to `ncurses_register_window_observer()` and `ncurses_register_window_remove_observer()` -- replaced by CEL_Observe auto-registration.

   ```c
       cel_has(NCurses_WindowConfig,
           .title = cel.title,
           .fps = cel.fps,
           .color_mode = cel.color_mode
       );
       /* SC5: Declare initial placeholder WindowState in composition.
        * Real values (COLS/LINES) are set by ncurses_window_frame_update() on first frame. */

       /* Bind entity to window lifecycle -- fires on_create observer.
        * Uses cels_lifecycle_bind_entity() because cel_lifecycle() does not exist
        * as a CELS composition verb. See ROADMAP SC5 update. */
       cels_lifecycle_bind_entity(NCursesWindowLC_id, cels_get_current_entity());
   }
   ```
   CRITICAL ordering: `cel_has(NCurses_WindowConfig, ...)` MUST come before
   `cels_lifecycle_bind_entity()` because the on_create observer reads the config from the entity.
   `cel_has(NCurses_WindowState, ...)` MUST also come before the lifecycle bind so the entity
   has both components when on_create fires.

   Note: `NCursesWindowLC_id` is a static variable created by `CEL_Lifecycle(NCursesWindowLC)`
   in the same TU. `cels_get_current_entity()` returns the entity being composed.
