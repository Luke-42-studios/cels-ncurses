- Replaced bridge-based flecs observer pattern with CEL_Lifecycle + CEL_Observe pattern in ncurses_module.c

## Decisions Made

1. **Lifecycle/observers in ncurses_module.c (not tui_window.c):** CEL_Lifecycle creates static variables (NCursesWindowLC_id) that must be visible to CEL_Compose for cels_lifecycle_bind_entity(). Placing all in the same TU eliminates cross-TU visibility issues.

2. **on_create does NOT set components (SC3):** Initial NCurses_WindowState is declared via cel_has in composition with placeholder zeros. The first frame of ncurses_window_frame_update() overwrites with real COLS/LINES. This avoids duplicate component-setting paths.

3. **cels_lifecycle_bind_entity() instead of cel_lifecycle():** cel_lifecycle() does not exist as a CELS composition verb. The manual API call achieves the same binding.

4. **ROADMAP SC5 already correct:** The plan anticipated needing to update ROADMAP SC5 from `cel_lifecycle()` to `cels_lifecycle_bind_entity()`, but the ROADMAP already had the correct text when Plan 02 was written.
