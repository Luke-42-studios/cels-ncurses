## Decisions Made
- **TUI_Input_use removed entirely:** The no-op stub and its TUI_Input config type were dead code -- no consumers reference them. Cleaned up now rather than deferring to a future phase.
- **CEL_System with cel_run pattern:** Both input and frame systems are queryless (no ECS components to iterate). The `cel_run { body }` pattern skips the registration pass and executes the body directly each frame.
- **flecs::flecs_static as INTERFACE dependency:** The bridge file (ncurses_ecs_bridge.c) directly includes `<flecs.h>`. Since cels-ncurses is an INTERFACE library (sources compile in consumer's context), consumers need flecs headers available. Adding it as an INTERFACE link propagates the include paths.
- **src/ as INTERFACE include directory:** The bridge header (ncurses_ecs_bridge.h) lives in `src/` rather than `include/`. Since tui_window.c includes it with `#include "ncurses_ecs_bridge.h"` (relative), the src/ directory must be on the include path for INTERFACE library compilation.
