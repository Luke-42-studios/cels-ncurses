# Milestone v1.0: Graphics API

**Status:** SHIPPED 2026-02-08
**Phases:** 1-5
**Total Plans:** 15

## Overview

Transform the cels-ncurses module from its current stdscr-based renderer with embedded app-specific widgets into a clean, low-level terminal graphics API. The build is strictly bottom-up: foundational types and color system, then drawing primitives, then panel-backed layers with resize handling, then the frame pipeline that orchestrates it all, and finally extracting app-specific code to leave behind a pure graphics backend suitable as a render target for cels-clay.

## Milestone Summary

**Key Decisions:**
- alloc_pair (not init_pair) for dynamic color pair management with LRU eviction
- wattr_set (not attron/attroff) for atomic attribute application
- WINDOW* borrowed not owned -- caller manages lifetime
- Scissor stack: silent early return on overflow/underflow (no assert)
- Panel-backed layers with extern globals (validated for INTERFACE library)
- werase (not wclear) for dirty clearing to avoid flicker
- ECS frame systems: FrameBegin at EcsPreStore, FrameEnd at EcsPostFrame
- stdscr reserved for input only, all drawing through TUI_DrawContext

**Issues Resolved:**
- stdscr-to-panels migration completed cleanly (Phase 3 risk)
- INTERFACE library extern globals validated (no per-TU duplication)
- Duplicate update_panels/doupdate removed from window loop (Phase 4)
- Module renderer stripped, pure graphics backend achieved (Phase 5)

- None
